# 1037Solo Public Roadmap

[English](./ROADMAP.md) | [简体中文](./ROADMAP.zh-CN.md)

This document tracks high-level external priorities for the 1037Solo ecosystem.

## Operating Principle

- Stable external communication;
- Clear product boundaries;
- Continuous delivery with measurable user value.

## Current Status (as of April 2026)

| Product | Status | Note |
|---|---|---|
| **StudySolo** | Beta | Closed beta in progress |
| **AISolo** | Beta | Closed beta in progress |
| **Classolo** | Planning | Intensive development starting May 2026 |
| **KitSolo** | Planning | In preparation |
| **OneThingSolo** | Planning | In preparation |
| **1037Solo Nav** | Paused | On hold |
| **Docs** | Active | Continuously maintained |

## Now (Q2 2026)

- Operate StudySolo and AISolo closed beta; collect user feedback and iterate on core workflows.
- Begin intensive Classolo development in May, focusing on lecture note-taking and review workflows.
- Advance planning and early prototyping for KitSolo and OneThingSolo.
- Consolidate AISolo (formerly Platform) as the unified AI agent platform with search, chat, and resource aggregation.
- Deepen StudySolo's AI Agent integration for planning, execution, and feedback loops in learning workflows.

## Next (Q3 2026 — Summer)

- Progressively develop KitSolo, OneThingSolo, and other planned products.
- Continue Classolo development toward feature-complete state.
- Expand StudySolo and AISolo beta scope based on feedback.
- Start progressive opening plans for API, CLI, and MCP capabilities.
- Strengthen product documentation and onboarding through [docs.1037solo.com](https://docs.1037solo.com).

## Later (Q4 2026 — After Fall Semester Starts)

- Target official public release for Classolo, KitSolo, and OneThingSolo.
- Strengthen cross-product integration across the Solo ecosystem.
- Introduce advanced AI agent templates and workflow automation in AISolo.
- Begin API beta access for selected integration partners.
- Explore CLI tooling for developer workflows including document sync and cloud environment configuration.

## Future (2027+)

- Continue expanding the 1037Solo platform around student daily needs and campus learning life.
- Introduce modules such as cafeteria atlas, 1037wiki, latest major-specific updates, and community/forum capabilities.
- Achieve MCP compatibility for AI-native tooling and agent ecosystem integration.
- Build a long-term ecosystem that empowers every HUSTer with practical, trustworthy campus intelligence.
- Evaluate sustainable business models and partnership frameworks.

## Update Policy

- This roadmap is reviewed and updated quarterly.
- Detailed implementation plans are managed in internal repositories.
