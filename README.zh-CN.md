# 1037Solo

[English](./README.md) | [简体中文](./README.zh-CN.md)

> 面向学习场景、学术效率与学生知识工作流的 AI 原生产品。

1037Solo 团队的 GitHub 官方主页仓库。

## 使命

1037Solo 致力于构建聚焦学习场景、学术效率与学生知识工作流的 AI 原生产品。

我们的目标是帮助学生把碎片信息转化为结构化理解、可执行产出与长期学习成长。

## 本仓库定位

这个特殊仓库（`1037Solo/1037Solo`）是团队在 GitHub 的官方公开落地页。

主要用于：

- 官方定位与对外沟通；
- 产品与仓库导航；
- 路线图与更新日志的公开发布。

本仓库不作为功能开发仓库使用。

## 运营透明度公告

1037Solo 目前由一名开发者独立维护，日常需兼顾学业与开发。项目暂无外部资金注入，服务器与 API 等费用均由个人承担。

关键运营说明：

- **数据库**：各项目数据存储基于 Supabase Free 版。连续 7 天无活跃访问后数据库可能自动暂停，如遇异常请联系我们。
- **AI 服务**：API 采用按量计费，当余额用尽时 AI 功能可能暂时离线。收到反馈后会及时补充额度。
- **使用资格**：暂无条件接入正式支付及营业执照，不支持直接购买开通。内测资格通过邀请码发放——请邮件联系团队获取。
- **持续运行承诺**：核心服务器与域名续费将持续进行。用户生成的数据（个人介绍页、笔记等）妥善存储不中断。

紧急联系：[contact@yusheng.email](mailto:contact@yusheng.email) 或 [AIMFllyYS@gmail.com](mailto:AIMFllyYS@gmail.com)

> *企业邮箱（support@1037solo.com）偶有收发异常，紧急事务请优先使用个人邮箱。*

## 平台能力开放方向

1037Solo 将逐步开放部分平台能力：

- **API 接入** — 面向集成场景的标准接口，复用知识聚合与模型推理管道；
- **CLI 工具** — 面向开发工作流的原生命令行套件，支持文档同步、构建检查与云环境配置；
- **MCP 兼容** — 兼容 Model Context Protocol，接入 AI 原生工具链与 Agent 生态。
